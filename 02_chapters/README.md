# 02_chapters — 章節切分

《三國演義》前 60 回，每回切成 2 個 .md 檔，共 **120 個**。

## 切分規則

- 以 `第 N 回` 標題為一級邊界
- 每回再切成大致等長的兩段（依字數中點與最近段落結尾切）
- 保留章節 frontmatter：書名、回數、編號、字數

## 檔名格式

```
c001.md ~ c120.md
```

奇數編號為該回上半，偶數為下半。例如：
- `c001.md` = 第 1 回上半
- `c002.md` = 第 1 回下半
- `c003.md` = 第 2 回上半
- ……

## 為什麼這樣切

| 設計 | 原因 |
|---|---|
| 每回切兩段 | 直接整回送 LLM 容易超出單次萃取的 token 預算 |
| 不切更細 | 切太細會把同一場戰役的人物分散在多個 chunk，後續關係不好合 |
| 保留章節邊界 | 三國回目本身是天然語意邊界，可直接做「某章節範圍內」的篩選 |
