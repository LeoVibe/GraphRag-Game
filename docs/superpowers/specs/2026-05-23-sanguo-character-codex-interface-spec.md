# 三國演義探險地圖：人物圖鑑 × 關係偵探 介面設計規格

伴隨文件：`2026-05-23-sanguo-character-codex-design.md`（總設計）
版本代號：`sanguo-character-codex-interface`
日期：2026-05-23
覆蓋範圍：三大區塊（主頁地圖 / 任務舞台 / 人物圖鑑）的視覺、互動、響應式、a11y 規範

## 1. 設計策略：三主題對應三區塊

沿用 v17 已實作的視覺主題系統，把 `body.style-*` class 從「使用者偏好」改為「區塊既定主題」：

| 區塊 | 對應主題 class | 為什麼 | 質感關鍵字 |
|---|---|---|---|
| 主頁地圖 | `body.style-bamboo` | 鳥瞰世界、慢節奏觀察 | 紙感、米白、墨綠、赭石、低彩度 |
| 任務舞台 | `body.style-warroom` | 對戰感、聚焦感 | 深墨藍、沙盤格、亮邊、戰術桌 |
| 人物圖鑑 | `body.style-classic` | 寶可夢卡冊、可愛但不幼稚 | 圓潤、淡彩、卡片、漂浮 |

切換區塊 = 切換 body class，**自動**換配色 / 字體配對 / 質感——孩子有「換場景」儀式感。

ui-ux-pro-max 推薦的 Claymorphism 元素**僅注入**升等動畫、卡片點擊回彈這類遊戲化瞬間，不全站套用（避免對國中生過幼稚）。

## 2. Design Tokens

### 2.1 字體

```css
--font-display: "Noto Serif TC", "Source Han Serif TC", serif;     /* bamboo 標題 */
--font-heading: "Noto Sans TC", system-ui, sans-serif;             /* warroom/classic 標題 */
--font-body:    "Noto Sans TC", system-ui, sans-serif;
--font-number:  "Baloo 2", "Noto Sans TC", monospace;              /* 進度、星等 */
```

### 2.2 字級階梯（4 級全站可選 0.875× / 1× / 1.125×）

| Token | px | line-height | 用途 |
|---|---|---|---|
| `--text-xs`   | 12 | 1.5 | 章節標籤、輔助 |
| `--text-sm`   | 14 | 1.6 | 卡片副文 |
| `--text-base` | 16 | 1.7 | 題幹、內文（兒童最小） |
| `--text-lg`   | 20 | 1.5 | 區塊標題 |
| `--text-xl`   | 28 | 1.3 | 題目主問句 |
| `--text-2xl`  | 36 | 1.2 | 進場慶祝、人物名 |

### 2.3 Semantic 色票（程式碼用 token 不用 hex）

```
--color-primary       主動作（解鎖、答對）
--color-secondary     次動作（提示、跳過）
--color-success       答對綠
--color-warning       提示橘
--color-danger        答錯紅（柔和）
--color-bg            主背景
--color-surface       卡片背景
--color-border        邊框
--color-text          主文字
--color-text-soft     輔助文字
--color-faction-wei   曹魏
--color-faction-shu   劉蜀
--color-faction-wu    東吳
--color-faction-lords 群雄
```

三主題 hex 對映（給設計核可，不出現在 component 代碼）：

| Token | bamboo | warroom | classic |
|---|---|---|---|
| primary | `#7C5E2C` 赭石 | `#F59E0B` 沙盤金 | `#F97316` 馬卡龍橘 |
| bg | `#F7F1E1` 宣紙 | `#1E293B` 墨藍 | `#FFF7ED` 暖白 |
| surface | `#FBF6E8` | `#334155` | `#FFFFFF` |
| text | `#3F2D14` 墨褐 | `#F8FAFC` 沙白 | `#1E293B` 深墨 |
| success | `#5B8B4A` 竹綠 | `#10B981` 翠 | `#16A34A` 亮綠 |
| danger | `#A14E3A` 赭紅 | `#EF4444` | `#EF4444` |
| 陣營 wei | `#3F6F8E` | `#60A5FA` | `#2563EB` |
| 陣營 shu | `#A14E3A` | `#FBBF24` | `#F97316` |
| 陣營 wu | `#5B8B4A` | `#34D399` | `#16A34A` |

對比要求：所有 surface ↔ text 配對 ≥ 4.5:1（AA）；主文字目標 ≥ 7:1（AAA），詳見第 6 章兒童 a11y。

### 2.4 圓角、陰影、動效

```
--radius-sm: 8px   --radius-md: 16px   --radius-lg: 24px   --radius-pill: 999px
--shadow-card  淺  --shadow-lift  升等浮起  --shadow-glow  解鎖光暈

--motion-tap:       120ms ease-out
--motion-state:     200ms ease-in-out
--motion-page:      320ms cubic-bezier(.2,.8,.2,1)
--motion-celebrate: 480ms spring(stiffness: 280, damping: 18)
```

全站尊重 `prefers-reduced-motion: reduce` → 所有 motion 降為 ≤ 80ms fade。

### 2.5 Spacing、Touch target

- Spacing：4 / 8 / 12 / 16 / 24 / 32 / 48（8pt 系統）
- Touch target：≥ 44 × 44 px；間距 ≥ 12 px；icon-only 控制必加 padding 不只依 visual size

## 3. 三區塊桌面 layout（≥ 1024px）

### 3.1 主頁地圖（bamboo）

橫向卷軸，12 個事件包像古地圖關隘節點，三狀態：已過 / 進行中 / 鎖。

頂部固定條：站名 + 圖鑑進度條 + 字級切換 + 設定。

點節點 → 右側面板（不是 modal）滑入：「本包學什麼 / 預計遇見 5 位人物 / 開始挑戰大鈕」。
鎖節點點下抖一下 + 提示「先完成上一關」。

### 3.2 任務舞台（warroom）

聚焦無雜訊。版面從上到下：
1. 頂部極簡導覽：「← 回地圖   2/5 題   💡 提示」
2. 題目主問句（`text-xl`，置中）
3. 線索卡（章節資訊）
4. 候選 4 chip / 2×2 網格（依題型而定）
5. 主 / 次 CTA（「交卷」「換題目」）
6. 迷你關係圖（沙盤格背景，提示用）

答後覆蓋層（從題目位置展開的全頁 overlay，spring 動畫）：
- 答對：「✨ 答對了！」+ 解釋 + 升等動畫 + 下一題 / 在地圖上看 / 回地圖
- 答錯：「再想想看，差一點 🌱」+ 你選的這個其實是 OOO + 正確位置閃爍 + 再試一次 / 看提示

### 3.3 人物圖鑑（classic）

頂部過濾：陣營 chip（全/魏/蜀/吳/群雄）+ 熟識度 chip（★★★/★★/★/?）
主區：依熟識度分組的卡片網格。
點卡片 → 翻面詳情（spring flip）：頭像、別名、陣營、個性條（基於 relationship category 比例）、「我為什麼認識他」時間軸、在關係圖中看他按鈕。

## 4. 四種題型 layout 差異

| 題型 | 主要互動 | 視覺特徵 |
|---|---|---|
| 誰是誰 / 他們什麼關係 | 4 chip 選 1 | 候選顯示頭像 + 名字 + 陣營色點 |
| 誰是同夥 | 4 頭像可多選 | 3 個預設選 + 「曹魏」標籤，找不屬於的 |
| 關係鏈 | 4 條路徑視覺化 | 路徑用迷你節點圖呈現，選最短 |
| 個性配對 | 拖拉 | 個性敘述卡 → 拖到人物頭像 |

## 5. 手機版 layout（< 768px）

設計原則：**手機是首要目標**，桌面只是「放大版」不是「複雜版」。

### 5.1 主頁地圖手機版

直向卷軸，節點垂直排列，每節點之間用「▼」連接線連續感。

### 5.2 任務舞台手機版（拇指區優先）

```
頂部安全距離（避瀏海/動態島）
題目主問句：置中、字級 text-xl、垂直留空白
線索卡：較小字
候選：2×2 網格，每格 ≥ 88×88 px（雙倍 touch）
        頭像 + 名字 + 陣營色點
底部固定動作條（高 64px，避 home indicator）：
  左下：「⤺ 換題」（次 CTA，視覺較弱）
  右下：「交卷 →」（主 CTA，右拇指可達）
```

答後覆蓋層用**全頁 sheet 從下滑上**，不用中央 modal。

### 5.3 人物圖鑑手機版

```
頂部：← 圖鑑 + 🔍 搜尋 + 🎚 篩選（icon-only 加 aria-label）
陣營 chip 橫滑列
分組摺疊區：★★★（3）/ ★★（9）/ ★（8）/ ?（40）
卡片網格：兩欄，每張 ≥ 160×200 px
點卡片：全頁覆蓋（非抽屜），底部固定 CTA
```

### 5.4 橫向手機

特例。任務舞台垂直空間是核心，因此**強制提示**「翻直拿手機體驗較好」。

### 5.5 響應式 breakpoint 策略

```
< 480px    手機（直向）      單欄、底部固定 CTA、抽屜改全頁
480-767px  大手機/直向平板  單欄但卡片更大
768-1023px 平板（橫向）     雙欄：主區 + 側邊細節
≥ 1024px   桌面            完整三欄、卷軸寬展開
≥ 1440px   寬桌面          側邊插畫裝飾、不增加密度
```

iPad 平板是最棒的形態，雙欄佈局 + 可選 1-4 鍵盤快捷答題。

## 6. 兒童 a11y 特別條款

ui-ux-pro-max 標準 a11y 全照做，**額外**：

| 條款 | 標準 | 為什麼為兒童特別重要 |
|---|---|---|
| 最小字級 | 16px base，14px 用於小選項 | 視力發展期 |
| 觸控目標 | ≥ 44×44 px、間距 ≥ 12 px | 手指控制力弱，誤觸率高 |
| 對比 | 主文 ≥ 7:1（AAA） | 注意力低、低對比立刻放棄 |
| 顏色不獨用 | 答對/錯永遠配 icon + 文字 | 色盲 + 兒童顏色語意未建立 |
| 動效強度 | spring overshoot ≤ 1.08 | 過大動效讓兒童分心 |
| 閃光禁忌 | 無 > 3Hz 閃爍、無高飽和紅↔黑切換 | 光敏性癲癇保護 |
| reduce-motion | 所有 spring → 80ms fade | 注意力障礙 / 暈眩體質 |
| 文字密度 | 單卡 ≤ 50 字、整頁 ≤ 200 字 | 兒童閱讀續航有限 |
| 音效（未來） | 預設關，要開要主動，每聲 ≤ 200ms | 公共場合自學 |
| 錯誤語氣 | 「再想想看，差一點 🌱」不「錯誤！」 | 介面要當朋友 |
| 進度可見 | 每答 1 題都看得到變化 | 兒童需即時回饋有動機 |

## 7. 友善設計（自學者一個人玩也不會卡）

### 7.1 Onboarding（第一次進站）

3 步輕觸引導（可跳過）：
1. 「歡迎來到三國世界 🏞」+ 亮點動畫指向第一個事件包
2. 「點亮的關卡可以挑戰」+ 第一個關卡微跳 1 次
3. 「答對會學到人物的故事」+ 圖鑑進度條閃光

全部 spring 動畫，孩子可隨時點掉繼續探索。

### 7.2 卡關保護（最重要）

| 情境 | 介入時機 | 行為 |
|---|---|---|
| 同題思考 > 30 秒 | 自動 | 線索區出現「想看一條提示嗎？」 |
| 連錯 3 題 | 自動 | 啟動提示模式（題下顯示一條相關邊高亮） |
| 連錯 5 題 | 自動 | 直接通關 + 「沒關係，這場我們先過去」 |
| 同包停留 > 10 分鐘 | 自動 | 「要不要先回主頁換個地方？」 |
| 進站第 2 天 | 自動 | 主頁顯示「歡迎回來！你上次認識到劉備 ★★」 |

### 7.3 進度感（隨時可見、隨時可慶祝）

- 頂部進度條全站可見：●●●●○○○○○○ 12/60 ★3
- 答對 → 卡片彈跳（scale 1 → 1.08 → 1）+ 進度條 +1 動畫
- 解鎖新人物 → 圖鑑頂部 toast：「新人物：諸葛亮加入圖鑑！」3 秒自動消
- 每解鎖 10 人物 → 中央慶祝動畫（彩帶 3 秒，可關閉）
- 完成事件包 → 全頁慶祝畫面 + 下一關預告

### 7.4 錯誤恢復原則

所有「我做錯了」情境給「再試一次」**而不是**「重新開始」——孩子永遠不會 lose progress。

## 8. 既有 v17 資產的處理

| v17 資產 | 處理 |
|---|---|
| `body.style-classic / bamboo / warroom` CSS | 保留，從「使用者偏好」改為「區塊既定主題」 |
| 字級切換「小 / 中 / 大」分段按鈕 | 保留位置（頂部工具列），對應 `--text-*` token 全站縮放 |
| 關係人物 filter chips | 保留設計語言，遷移到圖鑑頁的陣營篩選 |
| sanguo visual style 色票 | 對應到本 spec 三主題 hex 表 |

## 9. 驗收條件

| 項 | 標準 |
|---|---|
| 三主題視覺差異 | 切換區塊時，背景、卡片質感、配色明顯不同 |
| 對比 | 所有文字 contrast ≥ 4.5:1，主文 ≥ 7:1 |
| Touch target | 所有可點元素 ≥ 44×44 px，間距 ≥ 12 px |
| 字級切換 | 小/中/大 三段全站生效，無 layout break |
| reduce-motion | OS 設定開啟時，所有 spring 變 80ms fade |
| 手機任務舞台 | 主 CTA 在右拇指可達區（底部右側） |
| 卡關保護 | 連錯 5 題自動通關，永不 lose progress |
| 響應式 | 375 / 768 / 1024 / 1440 都不破版 |

## 10. 明確不做的事（YAGNI，沿用主 spec § 12）

- 黑夜模式：三主題已涵蓋 light / warm / dark 三層，黑夜模式留待未來
- 音效系統：留待未來
- 自訂主題：使用者不能改主題色，避免破壞設計一致性
- 動畫密度設定：只區分 normal / reduce 兩段
