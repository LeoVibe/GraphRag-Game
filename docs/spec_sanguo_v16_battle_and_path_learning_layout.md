# 三國演義探險地圖 V16：戰役與路徑學習版位重構規格

版本：`battle-and-path-learning-layout`

## 目標

V16 將 `戰役推理` 與 `關係路徑` 從「共用人物模式版位」改成各自符合學習任務的閱讀桌。三個模式保持同一套空間語法：

- 左側：選擇目標與設定條件。
- 中間上方：地圖看見關係。
- 中間下方：證據閱讀與分類。
- 右側：摘要、檔案、時間線與學習提示。

這樣孩子切換模式時不必重新理解介面，只要理解「這次要觀察什麼」。

## 人物探險修正

### 主要問題

V15 的人物探險左側上半部和下半部高度失衡，導致三位起點、搜尋與陣營篩選難以同時操作。`關係人物` 區塊一次展開太多長條卡片，資訊密度低，也無法快速掃讀。

### V16 調整

- 左側改成 58% / 42% 的閱讀比例。
- 上方只放 `三位起點`，保持最簡單的啟動方式。
- 下方整合 `選擇英雄`、陣營篩選、搜尋框與英雄列表。
- 陣營按鈕只保留 `魏 / 蜀 / 吳 / 其他`，點已選陣營會取消篩選。
- `關係人物` 改成精簡列表：
  - 每個分類用緊湊的橫列顯示。
  - 一列包含人物名、關係類型、章回與短描述。
  - 預設顯示 6 筆，避免佔滿整個下方閱讀區。

## 戰役推理

### 學習問題

一場戰役為什麼發生？誰做了關鍵決策？採取了什麼行動？最後造成什麼結果？

### 版位配置

- 左側：`選一場戰役`
  - 放戰役卡片與搜尋。
  - 不放人物關係分類，避免任務混淆。
- 中間上方：`戰役地圖`
  - 戰役或事件節點為中心。
  - 周圍顯示參與人物與相關事件。
- 中間下方：`四步推理板`
  - 使用 `原因 / 決策 / 行動 / 結果` 四個標籤。
  - 點選標籤後，下方只顯示該步驟的證據卡。
- 右側：`戰役檔案`
  - 顯示核心問題、雙方人物、勝負或轉折、關鍵章回。
  - 右側作為固定摘要，不承擔大量證據閱讀。

### 排除內容

戰役模式不顯示 `讓他受挫的人`、`合作援助`、`與他交戰的人` 等人物關係分類。這些只屬於人物探險。

## 關係路徑

### 學習問題

兩個人物之間，是透過哪些戰役事件、互動行為、身分背景被串起來？

### 版位配置

- 左側：`選兩個人`
  - 起點、終點、交換、清除與常用配對。
  - 關係條件只保留為路徑篩選，不和人物名冊混在一起。
- 中間上方：`路徑地圖`
  - 主線節點加上序號。
  - 非主線節點淡化，讓孩子看得出這是一條路。
- 中間下方：`路徑分段閱讀`
  - 每段以 `A -> B` 呈現。
  - 標明 `戰役 / 事件`、`互動`、`身分 / 背景` 三種證據類型。
  - 每段顯示章回、關係描述與共同事件或中間人線索。
- 右側：`路徑摘要`
  - 顯示起點終點、段數、中間人物、共同事件數。
  - 顯示三種證據類型的比例。
  - 找不到路徑時提供明確下一步。

## 視覺與互動原則

- 下方閱讀區要比 V15 更緊湊，避免大面積空白。
- 長文只放在右側或下方卡片內，不塞進左側控制區。
- 同一模式下只出現該模式有意義的資訊。
- 桌機避免多重巢狀捲動；手機保持單一主要抽屜操作。
- 按鈕與標籤維持 44px 左右可點擊高度，但文字和間距降低密度。

## 驗收條件

- HTML 內版本字串為 `v16-battle-and-path-learning-layout`。
- `戰役推理` 模式的右側標題為 `戰役檔案`，下方閱讀區為 `四步推理板`。
- `關係路徑` 模式的右側標題為 `路徑摘要`，下方閱讀區為 `路徑分段閱讀`。
- `人物探險` 左側可同時看見三位起點與下方英雄選擇入口。
- `關係人物` 區塊不再展開成大面積空白長條，改為精簡掃讀列表。
- 生成後輸出正式檔 `三國演義探險地圖.html` 與版本檔 `三國演義探險地圖_v16.html`。
